# proxyrasp — API документация

Ниже перечислены HTTP-запросы, которые обрабатывают контроллеры в этом проекте.

**GET /api/schedule/group/{groupId}** : Получить расписание группы
- Метод: `GET`
- URL: `/api/schedule/group/{groupId}`
- Заголовки: `Accept: application/json`
- Параметры:
  - `groupId` (path) — идентификатор группы (строка).
  - `start` (query) — начальная граница периода (строка). Формат `YYYY.MM.DD`.
  - `finish` (query) — конечная граница периода (строка). Формат `YYYY.MM.DD`.
  - `lng` (query) — язык / локаль (целое число). 1 - Русский
- Возвращает: `application/json` — JSON-массив объектов `SubjectDto` (строка в теле ответа):
  - `uid` — идентификатор события
  - `summary` — краткое название
  - `description` — описание
  - `location` — место проведения
  - `start` — время начала события
  - `end` — время окончания события
- Примечание: контроллер получает `.ics` от внешнего сервиса и конвертирует его в JSON.

Пример запроса:
`http://localhost:8080/api/schedule/group/565?start=2026.02.23&finish=2026.03.01&lng=1`

**GET /api/search/search** : Поиск
- Метод: `GET`
- URL: `/api/search/search`
- Заголовки: `Accept: application/json`
- Параметры:
  - `term` (query) — строка поиска. КИРИЛЛИЦА ЖЕЛАТЕЛЬНО ДОЛЖНА БЫТЬ В ДОПУСТИМОМ URI-ФОРМАТЕ (пример: ПЭ-231н => %D0%9F%D0%AD-231%D0%BD)
  - `type` (query) — тип поиска (строка):
        - group - группа;
        - student - студент;
        - person - персонал;
        - auditorium - аудитория (помещение).
- Возвращает: `application/json` — содержимое ответа внешнего сервиса поиска, формат определяется внешним сервером.

Пример запроса:
`http://localhost:8080/api/search?term=%D0%9F%D0%AD-231%D0%BD&type=group`

Пример ответа (JSON):

```json

[
    {
        "id": 565,
        "label": "ПЭ-231н",
        "description": "Факультет информационных технологий и компьютерных систем | Дневная",
        "type": "group"
    }
]

```
